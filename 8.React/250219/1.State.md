## 여기서 데이터를 표현하면?

```js
this.state = {
```

## state

"리액트를 사용한다!" 하면 상태(State) 개념을 꼭 알아야 함.
상태란? 데이터가 변할 때마다 화면이 자동으로 업데이트되는 값.
=> 렌더링이 되는 기준 값

`"State가 변하면 렌더링이 다시 일어난다!"`
즉, state는 화면을 바꾸기 위해서 컴포넌트 내부적으로 관리하는 데이터!

## setState란?

직접 값을 바꾸면 React가 변화를 감지 못함!  

```js
this.state = 2;
```

그래서 리액트가 값 변경을 감지할 수 있도록 함수를 제공함.  
그게 setState임.

## 왜 setState를 사용해서 값을 변경함?


```html
<script>
    let a = 0;
    handleClick(){
        a = a + 1;
    }
    render() {
        return (
            <div onClick={handleClick}>{a}</div>
        )
    }
</script>
```

state 값을 직접 변경해도 React는 이 변화를 감지하지 못함!
리액트는 setState를 호출해야만 "변경이 발생했구나!"를 인식함.

## state를 그냥 =로 할당해서 변경하면 안돼요?

나중에 리액트에 대한 배경과 지식을 알려드리면 좀 더 이해하기 수월할겁니다.  
왜 직접 할당해서 사용하는게 아니라, setState()라는 함수를 이용해서 값을 변경하는지.  

일단, 지금 설명드리는건 리액트에서는  

1. state를 직접 할당하면, 리액트가 렌더링을 감지하지 못합니다. => 변화를 감지하지 못함.
2. 리액트는 **VirtualDOM** 을 사용하여 변경을 감지
