# Git 권한 설정

## 발생한 문제

- 팀원들이 **Pull Request(PR)** 는 보냈지만  
  → **권한 설정 미비**로 인해  
  → **PR을 보낸 사람이 직접 머지까지 가능**한 구조였음

### 문제의 핵심

- **리뷰 & 승인 절차 없이 머지가 가능**했기 때문에  
  → 협업 구조상 **검토 없이 코드가 반영될 수 있었던 위험 요소** 존재
- 결과적으로 **"PR의 의미"**가 반감됨

---

### 되돌아보기

- **PR 체크리스트** 혹은 **권한 정책**을 초반에 확실히 설정했다면  
  → 팀원 간 **상호 검토**, **코드 품질 확보**, **소통 기회**가 더 명확했을 것
- 깃헙의 권한 설정(예: 브랜치 보호, 리뷰어 필수 설정 등)을 통해  
  → **"내가 PR 보내고 내가 머지하지 못하는 구조"** 를 강제할 수 있었음

---

### 개선 포인트

- 다음 프로젝트에서는 아래를 사전에 설정할 것:
  1. **브랜치 보호 규칙 설정**
     - 리뷰어 승인 1인 이상 필요
     - 직접 머지 불가 설정
  2. **팀별 깃 전략 수립**
     - PR 및 리뷰 프로세스 정의
     - 코드 머지 기준 명확화
  3. **초기 체크리스트 작성**
     - 권한 설정 포함한 초기 세팅 항목 정의 및 점검

---

### 결론

- 단순한 기술적 문제를 넘어  
  → **협업 구조의 신뢰성과 책임성을 높이는 핵심 포인트였음**
- **한 번 더 체크했더라면**, 더 체계적이고 안전한 협업 환경이 가능했을 것
